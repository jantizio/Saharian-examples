Messy single line textbox with no keyboard support

This is a messed up ARIA single line textbox implementation with a label and help text that:

  • doesn't provide the expected keyboard support;
  • exposes to assistive technologies an invalid relationship to its label and description, referencing objects that do not exist in the DOM.

Important: contenteditable is used as a dirty hack to avoid messing with the caret in a custom text field, which is "way too much more complicated" than it should be :(. Other hacks have been put in place to overcome (many) browser quirks.

Due to the fact that this specific implementation leverages the contenteditable attribute, browsers and assistive technologies might not exhibit the described issues.
